### Teacher Training (In Progress)
- Issue: Initial model size (hidden_dim=768, num_heads=12) caused GPU OOM on 23GB GPU
- Error: Attention mechanism tried to allocate 768GB due to flattened spatial dims
- Root cause: Sequence length = (batch, frames, height, width) flattened = huge attention matrix

#### Fixes Applied:
1. Fixed argument order: `model(latents, timesteps, audio_emb, ref_frame)` was wrong
   - Should be: `model(z, timesteps, audio_emb, ref_image)` where z is VAE latents
2. Added SpeechAE audio encoder: Raw audio (16000 samples) needs encoding to (B, T, 256) features
3. Reduced model capacity:
   - hidden_dim: 768 → 256
   - num_heads: 12 → 8
4. Reduced dummy dataset temporal frames: 8 → 4 frames per sample
5. Audio encoder frozen (no_grad) during training

### Key Technical Learnings:
- **Audio handling**: Model expects encoded audio features, not raw waveform
- **Attention memory**: O(seq_len²) cost is brutal; flattening spatial dims explodes memory
- **Dummy data loss**: ~0.5-0.95 is "good" on noise; don't expect < 0.1 until real data
- **GPU memory**: 23GB GPU can't handle large attention heads on spatiotemporal data
