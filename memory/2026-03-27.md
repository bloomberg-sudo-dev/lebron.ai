# Session: REST Training Pipeline Debugging (Mar 27, 2026)

## Context
Opemipo was training a Temporal VAE model on Runpod. Started with import errors and progressively fixed architecture/memory issues.

## Issues Encountered & Fixes

### 1. Missing Dataset Exports
- **Error**: `ImportError: cannot import name 'TalkingHeadDataLoader' from 'datasets'`
- **Root Cause**: `.gitignore` was blocking `datasets/` folder, so GitHub didn't have `talking_head_dataset.py`
- **Fix**: Created `datasets/__init__.py` with proper exports, then force-added `talking_head_dataset.py` to git

### 2. Parameter Name Mismatch
- **Error**: `TypeError: TemporalVAE.__init__() got an unexpected keyword argument 'latent_dim'`
- **Root Cause**: VAE was defined with `latent_channels`, not `latent_dim`
- **Fix**: Updated all three scripts (train_temporal_vae.py, train_teacher.py, inference.py) to use correct params

### 3. GPU Memory Issues
- **Error**: `torch.OutOfMemoryError: CUDA out of memory. Tried to allocate 32.00 GiB`
- **Runpod GPU**: 23.54 GiB total (RTX 4090 or similar)
- **Root Cause**: Dummy dataset was too large (16 frames × 64×64 pixels = too much memory)
- **Fix**: Reduced dummy dataset:
  - Frames: 16 → 8
  - Frame size: 64×64 → 32×32
  - Audio length: 2 sec → 1 sec
  - Batch size: capped at 2, then 1

### 4. VAE Architecture Mismatch
- **Error**: `ValueError: too many values to unpack (expected 2)` in forward()
- **Root Cause**: Complex VAE with FC layers expecting different shapes; mu/logvar computation was flawed
- **Decision**: **Complete revamp to SimpleTemporalVAE** — straightforward conv-based architecture
  - Removed FC layers entirely
  - Encoder outputs `latent_channels * 2` (split into mu and logvar)
  - Uses chunk() to cleanly split channels
  - Proper reparameterization trick
  - Simple MSE + KL divergence loss
  - Updated both train and validation loops

## Technical Decisions Made

1. **Simplified VAE**: Prioritized correctness + trainability over architectural complexity
2. **Dummy dataset sizing**: Empirically adjusted to fit in 23.5GB GPU with batch_size=1
3. **Git force-add**: Used `git add -f` to bypass `.gitignore` for essential Python modules

## Lessons Learned
- `.gitignore` can silently break training if it ignores essential modules
- VAE encoder shape mismatch is easier to debug with simpler architecture
- Small dummy datasets (32×32, 8 frames) can still use 1-2GB+ with conv layers
- Always test with `--epochs 2` on new hardware to catch memory/shape issues early
